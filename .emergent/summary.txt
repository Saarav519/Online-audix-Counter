<analysis>**original_problem_statement:** 
The user wants to build a clone of the Stock Count: Stock Take Opname mobile app as a web application named Audix Stock Management.

**PRODUCT REQUIREMENTS:**
- **Branding:** Use Audix branding with the provided logo and an eye-friendly color scheme.
- **Core Functionality:**
    - **Barcode Deletion:** Deleting a barcode should only affect the specific location it's being deleted from.
    - **Submit & Lock:** Locations must have a Submit option. Once submitted, they are locked and cannot be edited without manual authorization (requiring a confirmation popup).
    - **SKU Scanning Modes:** A toggle to switch between Single SKU mode (each scan adds 1 to the quantity of a single line item) and Multiple SKU mode (allows manual quantity entry).
    - **Product Master Control:** An option to allow scanning of only master products or to permit non-master products as well.
    - **Audio Feedback:** Different sounds for valid and invalid barcode scans.
- **Location Management:**
    - **Location Scanning Modes:**
        1.  **Pre-Assigned Mode:** Users can only scan locations from a pre-imported list. A feature to import this list via CSV is required.
        2.  **Dynamic Mode:** Scanning a new, unrecognized location code automatically creates it.
    - **Mode Separation:** Locations created/imported in one mode must not be visible in the other.
    - **Pre-Assigned View:** In this mode, locations should be displayed in a list/table view with an Open button. After completing one location, the app should auto-navigate to the next pending one.
- **Data Management:**
    - **Master Data Overwrite:** Uploading a new master data file should replace the old data.
    - **User Import:** An option to import user credentials along with the master data file.
- **Security & Access:**
    - **Settings Authentication:** Accessing the settings page and saving any changes must require user authentication.
    - **Daily Credential Changes:** An option should exist for daily user ID/password changes.
